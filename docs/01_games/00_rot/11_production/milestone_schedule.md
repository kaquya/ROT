# Milestone Schedule

Project: ROT  
Version: 1.1  
Document Type: Production  
Category: Development Planning  
Scope: Full Game Development  
Status: Core Reference

---

# 1. Overview

This document defines the development milestones for ROT.

Milestones divide the development process into structured phases that allow the team to track progress and evaluate project readiness at each stage.

Each milestone represents a meaningful step toward a complete playable game.

Milestones are designed to validate:

- core gameplay systems
- production pipelines
- content creation
- narrative integration
- technical stability

This schedule focuses on **development stages rather than specific calendar dates**, allowing flexibility depending on team size and production speed.

---

# 2. Development Structure

The project is divided into the following major milestones:

1. Project Foundation  
2. Core Systems Implementation  
3. Vertical Slice  
4. Full Production Start  
5. Content Expansion  
6. Alpha  
7. Beta  
8. Release Candidate  
9. Launch Preparation  

Each milestone concludes with a review phase to confirm readiness before proceeding.

---

# 3. Milestone 1: Project Foundation

Goal: Establish the technical and production foundations of the project.

Key objectives:

- project architecture defined
- core data structures implemented
- input system implemented
- basic state management system
- initial save system structure
- repository structure finalized
- documentation baseline completed

Deliverables:

- stable project framework
- development pipelines defined
- team workflow established

At the end of this milestone the project should be technically ready for gameplay system development.

---

# 4. Milestone 2: Core Systems Implementation

Goal: Implement the fundamental gameplay systems required for the core loop.

Key objectives:

- player controller system
- movement mechanics
- combat system prototype
- stealth system prototype
- interaction system
- inventory system
- crafting system
- UI system
- enemy AI prototype
- AI perception system

Deliverables:

- functional gameplay sandbox
- early test environment for mechanics

The focus of this stage is functionality rather than polish.

---

# 5. Milestone 3: Vertical Slice

Goal: Create a small but fully playable section of the game.

Scope:

- residential district environment
- connection to early downtown area
- several enemy encounters
- at least one puzzle
- safe zone implementation
- early narrative collectibles

Systems validated during this milestone:

- combat and stealth
- exploration loop
- Rot exposure mechanics
- enemy AI behavior
- UI systems

Deliverables:

- polished gameplay segment
- representative art style
- final tone and atmosphere

The vertical slice demonstrates the intended player experience.

---

# 6. Milestone 4: Full Production Start

Goal: Begin full scale environment and content development.

Key objectives:

- district level blockouts
- enemy roster expansion
- puzzle system expansion
- collectible placement framework
- narrative content writing
- audio system integration

Production pipelines for:

- environment assets
- enemies
- animation
- audio

must be fully operational.

---

# 7. Milestone 5: Content Expansion

Goal: Build the majority of the playable game world.

Key objectives:

- implement all major districts
- add additional enemy types
- integrate boss encounters
- expand narrative collectibles
- add environmental storytelling

Systems refined during this phase:

- Rot ecosystem mechanics
- encounter design
- resource distribution

This stage produces the majority of the game's content.

---

# 8. Milestone 6: Alpha

Goal: Achieve a fully playable game from beginning to end.

Characteristics of the Alpha stage:

- all districts implemented
- all core gameplay systems functional
- boss encounters implemented
- full narrative progression available

The focus shifts from content creation to testing and improvement.

Alpha builds are expected to contain bugs and incomplete balancing.

---

# 9. Milestone 7: Beta

Goal: Refine and stabilize the complete game.

Focus areas:

- bug fixing
- gameplay balancing
- performance optimization
- improved enemy behavior
- tuning resource economy

Beta builds should represent a nearly finished experience.

Major gameplay systems should no longer change significantly.

---

# 10. Milestone 8: Release Candidate

Goal: Prepare a near final version of the game.

At this stage the game should be:

- feature complete
- stable
- optimized

Remaining work includes:

- minor bug fixes
- audio and visual polish
- final balancing adjustments

Release candidate builds are used to verify the game is ready for launch.

---

# 11. Milestone 9: Launch Preparation

Goal: Prepare the game for release.

Tasks include:

- final bug fixes
- build verification
- packaging and distribution preparation
- marketing material preparation

The development team ensures the game is stable across all target platforms.

---

# 12. Milestone Reviews

Each milestone concludes with a review process.

Review goals include:

- confirming milestone objectives were met
- identifying technical risks
- evaluating gameplay quality
- determining readiness for the next development phase

This structured review process helps maintain project stability and production discipline.

---

# 13. Purpose of This Document

This document defines the development milestones for ROT.

By structuring the project into clear phases, the team can track progress, manage risk, and ensure the game evolves toward a stable and complete release.
